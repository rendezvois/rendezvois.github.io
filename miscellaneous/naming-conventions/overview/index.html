    Naming Conventions for File and Folder Names#

    Note: This page details the general guidelines that apply to all types of releases. See the other pages in the section for examples per category.

    Also note that these are merely recommendations. Feel free to alter them to your own preferences.

    File names are recommended to contain the following elements:#

    The items are arranged roughly in a sequential order. For example, conventionally, the Resolution comes before the Source in the filename.

    The name must not contain any non-ASCII characters. Use an online tool like this or this to validate it.

    Movie Title:#

    Either the original language title or English title from IMDb.

    If you want to include both titles, you can put an "AKA", but put the year first, otherwise it breaks metadata scraping in programs like Plex or Kodi.
    Example: Original.Title.2010.AKA.English.Title

    For foreign language titles, replace any accented characters with English equivalent ones.

    Examples: à with a | č with c | ô with o | æ with ae. Some online tools to help with this: One | Two | Three.

    Year:#

    The year mentioned on IMDb.

    For movie collection boxsets:

    • Year of release of the boxset, e.g. Martin Scorsese's World Cinema Project, No. 1 [2013]
    • Year range covering the movies in the boxset, e.g. David Cronenberg's Early Works [1966-1970]

    Cut / Version:#

    Any relevant cut or version name, like Unrated, Extended.Cut, Directors.Cut, English Dub, IMAX Version

    Check IMDb's "Technical specs" and "Alternate Versions" page to find out information about different cuts of a movie.

    For example, Deadpool 2's Technical specs and Alternate Versions pages.

    If IMDb does not have any relevant info, try these sources:

    • Wikipedia
    • Blu-ray.com
    • Movie-Censorship.com
    • Webpage of the Blu-ray / DVD distributor

    PROPER / REPACK / RERIP:#

    • REPACK should be used whenever you fix some error in your own earlier release, like changing the audio, adding a subtitle, fixing wrongly tagged language, etc.

      It is imperative that you do not reuse the initial filename unless it had zero snatches. Otherwise, it causes confusion when there are multiple files out in the wild with identical file names.

    • PROPER is similar to REPACK. It is generally used when replacing a release by a different user or group.

    • RERIP is similar to REPACK. It is generally used when you actually reencoded the video, like to fix a glitch or a new encode from a superior source.

    Resolution:#

    • For SD encodes from SD sources, resolution can be skipped.
    • For SD encodes from HD sources, use 480p or 576p as applicable.
    • For HD and UHD content, use 720p, 1080i, 1080p, 2160p as applicable.

    Source:#

    • SD encodes: DVDRip, NTSC.DVDRip, PAL.DVDRip, TVRip, VHSRip, WEBRip, LDRip
      • SD Blu-ray encodes: 480p.BluRay, 576p.BluRay
    • SD sources: DVD5, DVD9, NTSC.DVD5, NTSC.DVD9, PAL.DVD5, PAL.DVD9, WEB-DL
    • HD/UHD encodes: BluRay, UHD.BluRay, WEBRip, HDTV, UHDTV, HD-DVD
    • HD/UHD sources: Blu-ray, UHD.Blu-ray, Blu-ray.Remux, BD25, BD50, BD66, BD100, WEB-DL, HDTV, UHDTV, HD-DVD
      • For full blu-ray discs, region or distributor name can be added, e.g. GER.Blu-ray, GER.BD50, Kino.Lorber.BD66
      • For WEB-DLs, the name of the source should be added, e.g. AMZN.WEB-DL, NF.WEB-DL, DSNP.WEB-DL, HMAX.WEB-DL, HULU.WEB-DL, MUBI.WEB-DL, VIMEO.WEB-DL, ATVP.WEB-DL

    HDR Type:#

    Put the HDR type, e.g. HDR10, HDR10+, HDR10Plus, DV, DoVi, Dolby.Vision, HLG, HDR10.Dolby.Vision, HDR10.DV, HDR10+.DoVi

    Use HDR10Plus instead of HDR10+ because the + symbol gets stripped by some scripts that auto-upload to other places.

    How to check the mediainfo for the HDR type:

    Important: Ensure that you are not using an old version of mediainfo. Old versions of mediainfo can have incomplete or entirely missing HDR metadata.

    Some seedboxes come pre-installed with old versions. Latest mediainfo can be compiled and installed even on shared seedboxes.

    Video Format:#

    • Encodes: x264, x265, XviD, DivX
    • Discs: AVC, HEVC, MPEG-2, VC-1
      • MPEG-2 can be skipped on DVDs because they are always MPEG-2.
    • Remuxes: H.264, H.265, AVC, HEVC, VC-1, MPEG-2

    • WEB-DLs: H.264, H.265, HEVC, x264, x265

      • AVC and H.264 are the same format, but the convention is to use H.264 on WEB-DLs. Similarly, HEVC and H.265 are the same format, but H.265 is more frequently used on WEB-DLs.

      • x264 and x265 must only be used in WEB-DL file names if the mediainfo shows the encoding settings present. One school of thought is that WEB-DLs should always use H.264 and H.265 instead of x264 and x265 respectively, regardless of the presence of encoding settings.

    • HDTV captures are generally either H.264 or MPEG-2

    • HDTV encodes are generally x264

    Audio Format:#

    • Lossy: DD.5.1, DD-EX.5.1, DDP.5.1, AC3, AAC.2.0, DTS, DTS.5.1, DTS-ES.6.1

      Use DDP instead of DD+ because the '+' symbol gets stripped by some scripts that auto-upload to other places.

    • Lossless: DTS-HD.MA.5.1, DTS-HD.MA.2.0, TrueHD.5.1, FLAC.2.0, LPCM.1.0

      • Object-based formats: TrueHD.7.1.Atmos, DTS-X, DDP.5.1.Atmos, E-AC3.Atmos
    Alternate names of audio formats in mediainfo:

    Mediainfo uses different names for some audio formats than what is conventionally used in filenames.

    ❌ Mediainfo's "Format" field ✅ Use instead 💬 Comment
    DTS XLL DTS-HD.MA Not to be confused with DTS XLL X which is an object-based format like Dolby Atmos
    DTS ES XLL DTS-HD.MA DTS-HD MA track with a DTS-ES core
    DTS XLL X DTS-X Do not use DTS:X because colons are not a valid character on some OSes
    DTS XBR DTS-HD.HRA Has similar bitrates as DTS-HD MA, but is a lossy format
    MLP FBA TrueHD Add 5.1 / 7.1 based on mediainfo's channel info
    MLP FBA 16-ch TrueHD.7.1.Atmos None
    E-AC-3 JOC DDP.5.1.Atmos Use DDP instead of DD+. The '+' symbol gets stripped by some auto-upload scripts

    Group Tag:#

    Any tag you wish to use, in order to identify the release as yours.

    The tag must always be the at the end of the file name. Format should be -TAG, without any space before the -

    Do not use a tag which is already in use by another known group or releaser.
